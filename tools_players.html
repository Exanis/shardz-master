Players management

This module manage the players connection to the master server. It will handle managing the player’s sessions in the server.

server.tools.players.disconnect_player(player_token: str) → bool

Disconnect a player from the server.

Parameters

player_token – The player’s session id (as provided by try_login_player)

Returns

True if the player was disconnected, False otherwise

server.tools.players.require_user(token: str = Depends(authorization_key)) → str

Dependency to require an user to be logged in.

Parameters

token – The authorization key (automatically injected)

Returns

The player id

Raises

HTTPException – 401 If the token is invalid.

server.tools.players.try_login_player(player_id: str) → Optional[str]

Try to login a player. If the player is already logged in, returns None. Otherwise, creates a new session for the player and returns the session id.

The session ID is a random ID safe for the player to know ; it is not a static number linked to the user’s identity.

Parameters

player_id – The player id to login

Returns

The session id if the player is logged in, None otherwise
